01

What is Vim, Actually?

Vim (Vi IMproved) is a text editor built on the philosophy that your keyboard is faster than your mouse — and your hands should never leave the home row. It's been around since 1991, runs on every Unix-like system, works over SSH, and is the de facto editor in most EDA and embedded development environments.

Your classmates using gedit are clicking menus to do things you'll eventually do with two keystrokes. The upfront cost is real — but it pays off fast.

Why It Matters for Your Work
When Cadence Virtuoso, your remote compile server, or any EDA tool drops you into a terminal, gedit won't be there. Vim will. Being comfortable in Vim means you're never blocked by your environment.

Modes — The Core Concept

This is the most important thing to understand. Unlike gedit, Vim has modes. Keys don't just type characters — they do different things depending on which mode you're in. Once this clicks, everything else makes sense.

Golden Rule
When in doubt, press Esc. Multiple times if needed. It always gets you back to Normal mode, which is your safe ground.
03

Surviving Your First Session

Before anything else, you need to be able to open, edit, and exit Vim without panicking. Here's the absolute minimum to be functional.

1
Open a file

In your terminal: vim filename.v — Vim opens in Normal mode.

2
Start typing

Press i to enter Insert mode. The bottom bar shows -- INSERT --. Now type normally.

3
Stop typing

Press Esc to go back to Normal mode. The -- INSERT -- indicator disappears.

4
Save and quit

Type :wq then press Enter. :w = write (save), :q = quit.

CommandAction
:wSave (write) the file
:qQuit (only works if no unsaved changes)
:wqSave and quit
:q!Force quit, discard all unsaved changes
:w!Force save (overrides read-only in some cases)
ZZSave and quit (shortcut, no colon needed)
The Classic Trap
You open Vim accidentally and can't get out. Solution: press Esc a few times, then type :q! and hit Enter. You're free.

Search & Replace

CommandAction
/patternSearch forward for "pattern" (press Enter)
?patternSearch backward
nNext match
NPrevious match
*Search for word under cursor (forward)
#Search for word under cursor (backward)
:nohClear search highlights

Search and replace uses the :s (substitute) command, one of Vim's most powerful tools:

" Replace first 'wire' with 'reg' on current line
:s/wire/reg/

" Replace all occurrences on current line
:s/wire/reg/g

" Replace all occurrences in entire file
:%s/wire/reg/g

" Replace with confirmation prompt (y/n/a/q)
:%s/wire/reg/gc

" Case-insensitive replace
:%s/wire/reg/gi

" Replace only in lines 10 to 25
:10,25s/wire/reg/g

Working with Multiple Files
Open multiple files at once: vim file1.v file2.v testbench.v. Vim loads them as buffers. Use :ls to see them and :b <name> to switch. Much faster than opening separate windows.
08

Visual Mode & Selection

Visual mode lets you select text and then apply operators to the selection. There are three variants:

KeyModeBest For
vCharacter visualSelecting arbitrary text ranges
VLine visualSelecting whole lines (most common)
Ctrl+vBlock visualColumn editing — add same text to multiple lines

Once you have a selection, apply any operator: d to delete, y to yank, c to change, > to indent, < to un-indent, ~ to toggle case.

Block Edit — Extremely Useful for Verilog
Press Ctrl+v, select down multiple lines, press I (capital i), type your text, then press Esc. The text is inserted at the start of every selected line simultaneously. Perfect for adding // comments to multiple lines or aligning port declarations.
09

The . Dot Operator — Repeat Last Change

The dot command . repeats your last change. It sounds simple but it's one of the most powerful things in Vim. The moment something "clicks" with the dot operator is the moment Vim starts feeling genuinely fast.

Example: You want to add a semicolon to the end of 15 lines that are missing one.

  1. Go to the first line, press A (insert at end), type ;, then press Esc
  2. Move to the next line
  3. Press . — it repeats the entire "append semicolon" action
  4. Repeat: move, ., move, .
Design Principle
Think about your edits in terms of "what would I want to repeat?" and structure your actions accordingly. A good Vim workflow optimizes for repeatability with ..
10

Macros — Record & Replay Any Sequence

Macros let you record an arbitrary sequence of keystrokes and replay it. This is the ultimate automation tool for repetitive edits.

1
Start recording

Press q then a letter (e.g. a) to record into register a. The status bar shows recording @a.

2
Do your actions

Navigate, edit, move to the next line — do the exact sequence you want to repeat.

3
Stop recording

Press q again.

4
Replay

Press @a to run the macro. Press @@ to repeat the last macro. Use 50@a to run it 50 times.

Practical Example
You have 30 lines of signal declarations you need to wrap in assign statements. Record a macro that goes to start of line, types "assign ", goes to end of line, types " = 1'b0;", then moves down one line. Run it 29 more times with 29@a.

Splits & Windows

Vim can split into multiple panes, letting you view different files (or different parts of the same file) side by side — all in the terminal.

CommandAction
:sp filenameHorizontal split, open file
:vsp filenameVertical split, open file
Ctrl+w h/j/k/lMove focus to left/down/up/right pane
Ctrl+w wCycle through panes
Ctrl+w =Make all panes equal size
Ctrl+w qClose current pane
Ctrl+w rRotate panes
Common Workflow for IC Design
Open your design file on the left, testbench on the right: vim design.v → :vsp testbench.v. Reference them side by side without switching windows.

Your .vimrc — Configuration

The ~/.vimrc file is your personal Vim configuration. It's loaded every time Vim starts. Here's a solid starting configuration for IC/Verilog work:

" ~/.vimrc — IC Designer's Starter Config

" ─── Core Settings ─────────────────────────
set nocompatible            " Use Vim defaults, not Vi
set number                  " Show line numbers
set relativenumber          " Relative numbers (great for motions)
set cursorline              " Highlight current line
set showmatch               " Highlight matching brackets
set hlsearch                " Highlight search results
set incsearch               " Search as you type
set ignorecase              " Case-insensitive search...
set smartcase               " ...unless you use a capital letter

" ─── Indentation ───────────────────────────
set tabstop=4
set shiftwidth=4
set expandtab               " Use spaces instead of tabs
set autoindent
set smartindent

" ─── Usability ─────────────────────────────
set scrolloff=8            " Keep 8 lines visible above/below cursor
set wildmenu                " Better command completion
set ruler                   " Show cursor position
set backspace=indent,eol,start
syntax on                   " Syntax highlighting
filetype plugin indent on

" ─── Key Remaps ────────────────────────────
" Map jk to Esc — never reach for Esc again
inoremap jk <Esc>

" Clear search highlight with Enter
nnoremap <CR> :noh<CR>

" Save with Ctrl+s
nnoremap <C-s> :w<CR>
inoremap <C-s> <Esc>:w<CR>a

" Move between splits with Ctrl+hjkl
nnoremap <C-h> <C-w>h
nnoremap <C-j> <C-w>j
nnoremap <C-k> <C-w>k
nnoremap <C-l> <C-w>l

" ─── Verilog / HDL specific ─────────────────
autocmd FileType verilog,systemverilog setlocal tabstop=4 shiftwidth=4
The jk Escape Remap
Adding inoremap jk <Esc> to your .vimrc is a game-changer. Typing "jk" in insert mode instantly drops you back to Normal mode. Your fingers never have to leave the home row.

Vim for Verilog / IC Design

Vim handles Verilog natively with syntax highlighting. Here are patterns you'll use constantly in your work at EQSemi:

TaskHow to Do It in Vim
Rename a signal everywhere :%s/old_sig/new_sig/gc — with confirmation per match
Comment/uncomment a block Ctrl+v → select lines → I → type // → Esc
Jump to matching begin/end % on a bracket or begin/end
Indent a block of ports V to select lines → > to indent, < to un-indent
Duplicate a module port declaration yy to yank line, p to paste, then cw to rename
Go to a specific line number 42G or :42 — fast when compiler gives you an error line
Search for a module instantiation /module_name then n to cycle through
Add assign to 20 lines at once Record macro: qa → I → assign → Esc → j → q → 19@a
Enable Verilog Syntax Highlighting
Modern Vim has built-in Verilog support. Just open any .v or .sv file and syntax highlighting is automatic. If it's not working, add syntax on and filetype on to your .vimrc.
